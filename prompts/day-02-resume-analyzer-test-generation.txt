Act as a senior QA analyst testing Git-a-Job, a ReactJS + JavaScript resume SaaS.

Feature under test:
User uploads a resume, pastes a job description, runs AI analysis, and views ATS score, missing keywords, suggested improvements, and resume preview.

Known product risks:
- Resume parsing can fail for PDF/DOCX files
- Parsed sections can become corrupted or mixed
- AI suggestions must not invent candidate experience
- Missing keywords must not encourage false claims
- ATS score should update meaningfully when relevant resume content changes
- Authenticated flows may differ from public homepage flows
- Credits may be consumed for AI actions
- Re-analysis may use stale data if state is not handled correctly

Tech context:
- ReactJS frontend
- JavaScript ES2015+
- API routes/server-side logic
- GitHub workflow
- Jira Agile/Scrum process
- Future AWS Lambda/DynamoDB-style testing mindset

Generate a practical QA test suite with:

1. Happy path test cases
2. Negative test cases
3. Edge cases
4. Regression tests
5. Data integrity tests
6. AI safety / hallucination tests
7. Auth / credits tests
8. Accessibility checks
9. Performance checks
10. Automation candidates
