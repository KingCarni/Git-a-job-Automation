Issue Type: Task
Status: Done

Description:
We need a dedicated onboarding/FTUE ticket to support the new resume architecture direction.

Goal:
Create a first-time setup mode on the shared resume engine so new users can build their initial canonical resume/profile without being dropped directly into the normal job-aware flow.

Core direction:

- ONE shared resume engine
- TWO modes:1. first-time setup mode
2. normal job-aware mode

Important:

- Do NOT build a totally separate resume system.
- FTUE should be a mode/state of the same underlying resume engine.

First-time setup mode should be used when:

- new user
- no canonical structured resume exists yet
- no completed onboarding resume exists yet

Normal job-aware mode should be used when:

- user already has a canonical structured resume/profile
- user comes in from Jobs / Match / Apply Pack
- role/job context exists

Onboarding stance:

- The user's first resume/profile setup should be free.
- Do NOT charge during tutorial/onboarding for creating the initial canonical resume/profile.
- Paid per-role actions remain in the later jobs/apply workflow.

Suggested flow:

1. First-time upload/import
2. Guided resume setup using setup mode
3. Initial ATS analysis on structured state
4. Persist canonical structured resume/profile snapshot
5. Route to Account/Profile for optional cleanup
6. Route into Jobs/Match once setup is complete

Acceptance:

- First-time users enter setup mode instead of the normal job-aware resume flow
- Setup mode uses the same resume engine/components, not a separate system
- Initial resume/profile setup is free
- Completing setup produces the canonical structured resume/profile used by ATS and matching
- Returning users with a canonical resume/profile bypass setup mode and use the normal job-aware path
- Product messaging/tutorial copy clearly explains the Git-a-Job workflow: build a strong base resume once, then tailor per role later

Why this matters:

- Improves first-use clarity
- Avoids charging during tutorial/onboarding
- Creates a better foundation for ATS/Profile parity and later jobs matching
- Prevents long-term product drift from multiple divergent resume systems
