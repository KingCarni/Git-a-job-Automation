Issue Type: Task
Status: Done

Description:
After the cover-letter template system is cleaned up, we need a focused hardening pass for preview/print/export parity.

Problem

Even with a cleaner layout system, cover letters will still feel broken if preview, print preview, and exported output do not match closely enough. Cover letters are short documents, which makes spacing bugs, alignment drift, and awkward whitespace much more visible than in resumes.

Goal

Stabilize the curated cover-letter layout set across:

- on-page preview
- print preview
- export / downloaded PDF

This is a hardening pass, not a broad design rewrite.

Depends on

This ticket should follow the implementation/cleanup pass for the cover-letter layout system.

Scope

- Audit preview, print, and export behavior for the supported cover-letter layouts
- Fix common parity issues such as:- header alignment drift
- sender/date/recipient block spacing mismatches
- greeting/body/closing spacing inconsistencies
- long name or long company wrapping problems
- awkward top/bottom whitespace
- clipping/overflow in PDF or print output
- page-break issues for unusually long letters
- Validate a representative color-scheme sample on top of the kept layouts
- Ensure compatibility/fallback styles do not break export

Recommended test set

- each supported cover-letter layout with one neutral color scheme
- one stable layout with several representative color schemes
- short letter content
- medium/normal letter content
- unusually long letter content
- long sender name / long recipient/company values

Likely files

- `src/components/CoverLetterGenerator.tsx`
- `src/app/api/render-pdf/route.ts`
- `src/app/api/resume-pdf/route.ts`
- any print/export helper only if necessary

Acceptance criteria

- No major preview vs print vs export mismatch on the supported cover-letter layouts
- No obvious clipping, broken spacing, or layout collapse in representative export cases
- Long names/company fields wrap acceptably
- Longer letters degrade gracefully without obvious broken pagination
- Color schemes remain visually consistent in export for representative samples
- Cover-letter output feels intentional and stable enough to ship

Non-goals

- Do NOT reopen resume styling work
- Do NOT broaden the template set again
- Do NOT chase edge-case perfection before the main parity issues are fixed

Why this matters

The cover-letter surface is currently the weaker half of the document pair. Even a good layout cleanup will still underwhelm if export stability is shaky. This pass makes the curated system believable in real use.
